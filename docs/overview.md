# Обзор Redpolitika

**Redpolitika** — open-core сервис для проверки и автоматической правки текста по правилам редакционной политики (редполитики), заданным в YAML.

Аналог «Главреда», но под любую редполитику. Не привязан к русскому языку — подходит для любого языка, где работает RE2-совместимая регулярка или список слов.

## Проблема

Редакции, издательства, пресс-службы и контент-команды тратят ресурсы на вычистку текстов от:

- нарушений редполитики (тавтологии, штампы, канцеляризм)
- нежелательных оборотов, обсценной лексики
- неправильных типографских символов
- стилистических погрешностей

Существующие инструменты либо привязаны к одному языку, либо являются SaaS с непрозрачными тарифами.

## Решение

Redpolitika — self-hosted сервис, который проверяет текст по вашим правилам:

- **YAML-правила** — декларативный формат с композируемым деревом detect/fix
- **Независимость от языка** — работает с любым языком через regex / wordlist
- **Self-hosted** — полный контроль над данными, без отправки текстов вовне
- **Open-core (BSL)** — ядро открыто, enterprise-функции проприетарны
- **Плагины** — расширение через gRPC для LLM, NER, POS

## Как это работает

```
YAML-правила → [Загрузка] → Компиляция детекторов → API-сервер
                                                      ↓
Текст → POST /api/analyze → Применение правил → Флаги + оценки
                                                      ↓
WebSocket /ws/live ← Принятие/отклонение ← UI (CodeMirror 6)
```

1. Вы описываете правила редполитики в YAML-файлах
2. Redpolitika загружает, валидирует и компилирует их при старте
3. Текст отправляется через REST API или WebSocket
4. Сервер применяет дерево детекторов, возвращает флаги
5. Фронтенд показывает флаги инлайн в редакторе CodeMirror 6
6. Редактор принимает/отклоняет флаги, применяет автоправки

## Стек

| Компонент | Технология |
|-----------|-----------|
| Backend | Go, Echo, Uber FX (DI), DDD / ports-and-adapters |
| Frontend | Next.js (static export), CodeMirror 6, Tailwind, shadcn/ui |
| Database | SQLite (default) / PostgreSQL opt-in |
| Plugin system | HashiCorp go-plugin, gRPC |
| Rules | YAML, deep-merge слоёв, RE2-совместимые regex |

## Ключевые возможности

- **Композируемые правила** — 20+ методов детекции и 15+ методов фиксов
- **Два скоринга** — чистота и читаемость (0–10), нормализация на 100 слов
- **Inline-подавление** — отключение правил на участках текста через HTML-комментарии
- **WebSocket live** — флаги в реальном времени по мере набора текста
- **Extension points** — плагины для кастомной логики (LLM, NER, POS)
- **Слои правил** — base → project → override с deep-merge по id
